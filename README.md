# BMAD Initializer

This repository contains the `install.sh` script, an interactive utility designed to globally install and configure the `bmad-init` CLI tool. The `bmad-init` tool allows you to quickly and easily initialize the **BMAD-METHOD** environment in any project.

## 🚀 About the `bmad-init` CLI

The `bmad-init` command automates the configuration of your workspace to use BMAD tools. It reads your preferred settings configured during installation to set up modules, tools, and your Git repository correctly.

### Interactive Installation Features

The `install.sh` script provides a 100% interactive installation experience:

1. **Prerequisites Verification & Installation**: Automatically checks if `node`, `npx`, and `git` are installed. If missing, it offers to automatically install them via Homebrew (macOS) or APT (Linux).
2. **Tools Selection**: Prompts you to choose which BMAD tools you want to use by default (e.g., `antigravity`).
3. **Git Preferences**: Asks whether the initializer should automatically run `git init` on new projects.
4. **Git Identity**: If Git initialization is enabled, it asks for your default `user.name` and `user.email` (auto-detecting your global Git config as a fallback) so that every BMAD initialization uses the correct author info.

*(Your choices are saved in `~/.bmad-init-rc` and applied whenever you run `bmad-init`).*

---

### What `bmad-init` does (step by step)

1. **Prerequisites check**: Ensures that `node` and `npx` are installed on your machine.
2. **BMAD-METHOD Installation**: Executes the installation command via `npx` based on your chosen tools directly in the current directory.
3. **Git Initialization (Optional)**: If you opted for it during installation, it autonomously initializes a new repository (`.git`).
4. **Git Configuration (Optional)**: Configures the Git user email and name for the project based on your installer inputs.
5. **Saving configuration (Optional)**: Specifically adds the folders generated by BMAD to Git and creates an initial commit with the message `chore: Init project with BMAD-METHOD`.

## 🛠️ Prerequisites

Before using this tool, ensure you have the following installed on your machine:

- [Node.js](https://nodejs.org/)
- `npx` (usually included with Node.js and npm)
- [Git](https://git-scm.com/)

## 🏃 Installation & Usage

### 1. Installation

First, clone this repository and run the installer script:

```bash
chmod +x install.sh
./install.sh
```

*(Note: The `install.sh` script will run an interactive setup. At the very end, it might ask for your user password `sudo` to copy the `bmad-init` command globally to `/usr/local/bin/`).*

### 2. Usage

Once installed and configured, simply navigate to any of your projects where you want to initialize the BMAD environment, and run the command:

```bash
cd /path/to/your/project
bmad-init
```

Your project will then be instantly configured using your preset preferences!

### 3. Uninstallation

If you wish to remove the `bmad-init` tool and its configuration from your system, simply run:

```bash
chmod +x uninstall.sh
./uninstall.sh
```

*(This will safely remove the CLI command from `/usr/local/bin` and delete your saved preferences).*
